# 领导经营数据看板 — 设计方案

## 定位

为领导层提供一个**无需登录、可直接访问**的经营指标看板页面，展示中台现有数据（鲜果产量、出勤率、AKP采收密度）。

访问地址：`http://<server>/leader.html`

---

## 实现方式

不新建独立项目，**在现有 `report-collection-portal` 上扩展**，只新增文件，不改动现有功能。

---

## 新增文件

### 1. 后端接口

**文件**：`backend/app/api/public.py`

新增路由 `GET /api/public/leader`，无需鉴权，一次返回所有看板所需数据：

```json
{
  "data_date": "2026-06-01",
  "att_data_date": "2026-06-01",
  "kpis": {
    "today_production_ton": 234.5,
    "mtd_production_ton": 4521.0,
    "today_attendance_rate": 87.3
  },
  "production_trend": [
    { "date": "2026-05-03", "ton": 210.2 },
    ...
  ],
  "production_by_site": [
    { "site": "WPP", "today_ton": 120.1, "mtd_ton": 2300.0 },
    ...
  ],
  "attendance_trend": [
    { "date": "2026-05-03", "rate": 85.0 },
    ...
  ],
  "attendance_by_site": [
    { "site": "WPP", "rate": 87.3, "present": 320, "total": 366 },
    ...
  ],
  "akp_by_division": [
    { "division": "DIV 1", "akp": 83.2 },
    ...
  ]
}
```

**数据来源**：

| 指标 | 模型 | 字段 |
|---|---|---|
| 鲜果产量 | `DwdAgriHarvestDaily` | `actual_kg`, `mtd_actual_kg` |
| 出勤情况 | `DwdAgriAttendanceDaily` | `own_total`, `contractor_total`, `total_present` |
| AKP密度 | `DwdAgriAkpDensityDaily` | `akp_percent`, `division` |

查询范围：最近 30 天（`report_date >= today - 30 days`）

---

### 2. 注册路由

**文件**：`backend/app/main.py`（修改）

```python
from app.api.public import router as public_router
app.include_router(public_router, prefix="/api")
```

---

### 3. 前端看板页面

**文件**：`frontend/public/leader.html`

独立 HTML 文件，使用 ECharts CDN（无需 npm install / 重新构建）。

**页面结构**（从上到下）：

```
┌────────────────────────────────────────────────┐
│  [公司名]  经营数据门户        数据截止：2026-06-01  │
└────────────────────────────────────────────────┘

┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
│ 当日产量  │  │ 月累计   │  │ 今日出勤率│  │ 数据区间  │
│  234 吨  │  │ 4,521 吨 │  │  87.3%   │  │  近30天  │
└──────────┘  └──────────┘  └──────────┘  └──────────┘

┌─────────────────────┐  ┌─────────────────────┐
│  近30天鲜果产量趋势   │  │  各园区产量对比       │
│  [折线图]            │  │  [柱状图：当日/月累计]│
└─────────────────────┘  └─────────────────────┘

┌─────────────────────┐  ┌─────────────────────┐
│  近30天出勤率趋势     │  │  各园区出勤情况       │
│  [折线图]            │  │  [柱状图：出勤率%]    │
└─────────────────────┘  └─────────────────────┘

[若 AKP 数据存在，追加：]
┌─────────────────────────────────────────────┐
│  各小区近期 AKP 采收密度（%）[柱状图]           │
└─────────────────────────────────────────────┘
```

**图表说明**：

| 图表 | 类型 | X轴 | Y轴 | 备注 |
|---|---|---|---|---|
| 近30天产量趋势 | 折线图 | 日期 | 吨 | 全公司合计 |
| 各园区产量对比 | 柱状图 | 园区 | 吨 | 两组柱：当日/月累计 |
| 近30天出勤率趋势 | 折线图 | 日期 | % | 全公司合计 |
| 各园区出勤情况 | 柱状图 | 园区 | % | 含出勤人数/总人数标注 |
| AKP各小区 | 柱状图 | 小区名 | % | 数据不存在则不显示 |

**技术细节**：
- ECharts 5 via jsDelivr CDN
- 纯 Vanilla JS，无框架依赖
- 调用 `/api/public/leader`（Vite 代理转发到 FastAPI）
- 数据为空时显示占位提示，不报错
- 页面每 10 分钟自动刷新一次

---

## 开发顺序

1. 写 `public.py` 后端接口并验证返回数据
2. 在 `main.py` 注册路由
3. 写 `leader.html` 并在浏览器中验证图表
4. 检查无数据场景的容错显示
