## 五、开发机用途

开发机当前只用于内网测试：

```text
http://192.168.12.136/report-portal/indicator
```

它不能直接作为 Teams Bot 正式回调地址，因为：

1. `192.168.12.136` 是内网 IP，微软 Teams 云服务访问不到。
2. 当前开发机访问方式是 HTTP，不是 HTTPS。
3. Teams Bot Messaging endpoint 必须是公网 HTTPS。

Teams Bot 真联调仍建议使用阿里云公网服务器：

```text
https://julongtongchuan.icu/report-portal/api/teams-bot/messages
```

## 六、开发机待确认事项

1. 前端是否已完成构建：

```bash
cd /opt/report-portal/frontend
npm ci
VITE_API_BASE_URL=/report-portal/api npm run build
```

2. 前端产物是否已同步：

```bash
sudo mkdir -p /var/www/report-portal
sudo rsync -a --delete dist/ /var/www/report-portal/
```

3. nginx 是否已配置：

```text
/etc/nginx/sites-available/report-portal.conf
/etc/nginx/sites-enabled/report-portal.conf
```

4. 内网访问是否可用：

```bash
curl http://192.168.12.136/report-portal/api/health
curl -I http://192.168.12.136/report-portal/indicator
```
