# 使用现有域名部署 Report Portal

当前决定：不再单独申请新域名，直接挂到现有同传域名下面。

```text
现有域名：https://julongtongchuan.icu
报表前端：https://julongtongchuan.icu/report-portal/indicator
报表后端：https://julongtongchuan.icu/report-portal/api/
Teams Bot endpoint：https://julongtongchuan.icu/report-portal/api/teams-bot/messages
```

## 一、不要动同传已有路径

同传服务正在使用这些路径，部署报表系统时不要覆盖：

```text
/api/
/ws/
/bot-api/
/api/messages
```

报表系统统一放到：

```text
/report-portal/
```

## 二、服务器上建议的运行方式

```text
报表后端：127.0.0.1:18000
报表前端静态目录：/var/www/report-portal
报表项目目录：/opt/report-portal
PostgreSQL：建议用独立容器 report-portal-postgres，宿主机端口用 15432
```

这样不会抢同传原来的 Java、Bot、nginx 路径。

## 三、后端生产 .env 关键项

部署到服务器后，后端 `.env` 至少要有这些：

```env
DATABASE_URL=postgresql+psycopg://portal:<数据库密码>@127.0.0.1:15432/report_portal
STORAGE_DIR=storage
ALLOWED_ORIGINS=https://julongtongchuan.icu

TEAMS_PORTAL_URL=https://julongtongchuan.icu/report-portal/indicator
TEAMS_BOT_APP_ID=72ba3fa0-13ee-4386-9606-3c5a011a41e8
TEAMS_BOT_TENANT_ID=49306cd1-4f6e-45ae-9eff-59a1bd8936d2
TEAMS_BOT_APP_PASSWORD=<已生成的 Client secret value>
TEAMS_BOT_VALIDATE_INCOMING=true
TEAMS_BOT_NAME=Report Portal Bot
```

注意：本地可以用 `TEAMS_BOT_VALIDATE_INCOMING=false` 调试；部署到公网 HTTPS 后要改成 `true`。

## 四、Teams Developer Portal 要填什么

在 `Report Portal Bot` 的 Configure 页面，Endpoint address 填：

```text
https://julongtongchuan.icu/report-portal/api/teams-bot/messages
```

不是本地 `127.0.0.1`，也不是根路径 `/api/teams-bot/messages`。

把这个文件里的内容加到同传现有 HTTPS server block 里面：


## 六、部署后怎么检查

先检查前端：

```bash
curl -I https://julongtongchuan.icu/report-portal/indicator
```

再检查后端：

```bash
curl https://julongtongchuan.icu/report-portal/api/health
```

如果后端通了，再去 Teams Developer Portal 保存 Bot endpoint。

## 七、Teams 联调顺序

1. 先确认网页能打开：`https://julongtongchuan.icu/report-portal/indicator`
2. 再确认后端健康检查能打开：`https://julongtongchuan.icu/report-portal/api/health`
3. 在 Teams Developer Portal 保存 Bot endpoint。
4. 把 Bot 安装到目标频道或个人聊天。
5. 在目标频道里 `@Report Portal Bot` 发一条消息，或在个人聊天里给 Bot 发一条消息。
6. 回系统 `配置管理 / 通知配置 / 指标 Bot 定时通知`。
7. 点击刷新目标。
8. 给每个指标选择对应 Teams 目标、时间、启用。
9. 先点一个指标的测试发送，确认 Teams 能收到。

## 八、我继续部署前还需要你给什么

只需要现有域名这台服务器的 SSH 登录信息：

```text

用户名：
登录方式：密码或私钥
是否有 sudo/root 权限：
```

拿到后就可以开始把报表系统部署到 `https://julongtongchuan.icu/report-portal/`。
