设计原则

① 宽表优先，一表对一报

每张 DWS 表对应一张业务报表。所有维度字段 + 指标字段放同一行，前端展示无需 JOIN。

② 派生指标在此计算

完成率、出油率、损失率等派生指标由 DWS ETL 计算并存储，ADS/前端直接读取。

③ 窄表透视为宽列

DWD 的 work_type / period_type / chemical_code 等维度列，在 DWS 展开为独立列（pruning_ha / daily_cpo_kg 等）。

④ 今日 + MTD 同行

工厂生产表将 DWD 的 today/monthly 两行合并为一行，daily_* 和 mtd_* 前缀区分周期。

⑤ 粒度与报表对齐

粒度由报表决定（日×园区×小区 / 日×工厂 / 月×园区×地块），不随意升维或降维。

⑥ 刷新策略：全量替换

每次 ETL 按 (report_date, site, …) 做 UPSERT，不追加历史。DWD 是权威来源，DWS 可重算。
