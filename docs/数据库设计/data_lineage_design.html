2. 来源追踪

三种数据录入方式，追踪方式不同
来源类型source_typesource_batch_ids触发写入方式
文件上传 file_upload ["B2025060301"] 上传 Excel → 解析 → 写 DWS，可追溯到原始文件
手动录入 manual_entry []（空） 网页表单直接录入，无批次来源，通过 updated_at 知道什么时候录的
外部同步 external_sync []（空） 从 SAP / 外部数据库同步，通过 DataSourceConfig 配置
覆盖优先级：同一行数据（同日期+公司+园区+小区）有多个来源时，
file_upload 覆盖 manual_entry（文件解析的比手录的更可信）；
external_sync 是否覆盖由各表业务决定，需配置后明确。

3. 质量检查规则

数据写入 DWS 时自动执行，结果写入 quality_status / quality_message

检查分三类，按严重程度设 error / warning / ok：

① 格式检查（解析阶段，所有表通用）

检查项规则结果
日期字段必须是合法日期，且不能是未来日期error
园区/小区名必须存在于 dim_site / dim_divisionwarning（新园区可能未录入）
数值字段不能是文字（如 "N/A"、"-"），解析后为 null 的记录要标注warning
必填字段report_date、company_code、site 不能为空error

② 数值范围检查（按业务表，可配置）

表字段正常范围结果
铲果产量daily_actual_kg≥ 0，单小区日产量通常 < 500,000 kgwarning
铲果产量mtd_actual_kg ≥ daily_actual_kgMTD 不能小于当日产量error（逻辑错误）
铲果产量bbc_achievement_pct0% ~ 200%，超出可能是 BBC 填 0 导致warning
出勤h_attendance_pct / m_attendance_pct0% ~ 100%error
POM 生产daily_cpo_oe_pct（OE%）18% ~ 28%（正常出油率范围）warning
POM 生产daily_cpo_ffa_pct（FFA%）0% ~ 15%warning
KCP 生产daily_pko_extraction_pct40% ~ 55%warning
精炼daily_yield_pct（得率）85% ~ 100%warning
出勤率rainfall_mm（日降雨）0 ~ 200 mm，超出疑似单位错误（mm vs cm）warning

③ 完整性检查（批次级别）

检查项规则结果
小区覆盖本月已有 N 个小区有数据，本次上传只有 M 个（M < N），疑似漏报warning，标注缺失的小区名
工厂数据完整性POM 有今日数据但无 MTD 数据（两者应同时存在）warning
日期连续性同一园区跳过某几天没有数据（如 6/1 有，6/3 有，6/2 缺）warning（仅提示，不阻断）
quality_status 写入逻辑：
任何一项 error 检查失败 → error（数据不可信，前端展示红色标记）
有 warning 但无 error → warning（数据可用但需关注，前端展示黄色）
全部通过 → ok（正常展示）

error 不阻断写入，数据仍然写入 DWS，但前端查询时可过滤掉 error 行，避免影响汇总统计。

实现位置（代码层面）

  • 1
    解析后（parse_file 之后） — 格式检查
    在 parser.py 的 load_dwd_records() 内，每条记录写入前做格式和必填校验，结果写到 DWD 的 quality_status
  • 2
    写入 DWS 时 — 数值范围检查 + 逻辑一致性检查
    DWS ETL 函数计算完派生指标后，对照范围规则校验，结果写到 DWS 的 quality_status / quality_message
  • 3
    批次完成后 — 完整性检查
    整个批次解析完成后，对比本月已有数据检查小区是否有遗漏，结果写到 upload_batches.remark 或独立展示

4. 追溯查询示例

给定一条 DWS 数据，如何找到它的来源和质量问题

查某行数据的来源批次和质量

-- 查铲果产量 DWS 数据，同时看来源和质量
SELECT
    report_date, company_code, site, division,
    daily_actual_kg, mtd_actual_kg,
    bbc_achievement_pct,
    source_batch_ids,       -- ["B2025060301"]
    source_type,            -- "file_upload"
    quality_status,         -- "warning"
    quality_message,        -- "BBC目标为0，完成率无法计算"
    updated_at
FROM dws_agri_harvest_daily
WHERE report_date = '2025-06-03'
  AND company_code = 'PTSI';

-- 用 source_batch_ids 追溯到原始文件
SELECT
    b.batch_no, b.report_name,
    b.uploader, b.created_at AS upload_time,
    f.original_filename
FROM upload_batches b
JOIN upload_files f ON f.batch_id = b.id
WHERE b.batch_no = 'B2025060301';

查今天所有质量异常的数据

-- 汇总今日所有警告和错误
SELECT
    'dws_agri_harvest_daily' AS table_name,
    site, division,
    quality_status, quality_message,
    source_type, updated_at
FROM dws_agri_harvest_daily
WHERE report_date = CURRENT_DATE
  AND quality_status != 'ok'

UNION ALL

SELECT
    'dws_factory_pom_daily',
    site, '',
    quality_status, quality_message,
    source_type, updated_at
FROM dws_factory_pom_daily
WHERE report_date = CURRENT_DATE
  AND quality_status != 'ok'

ORDER BY quality_status DESC, updated_at DESC;

查手动录入的数据

-- 审计最近 7 天手动录入的记录
SELECT
    report_date, company_code, site, division,
    daily_actual_kg,
    updated_at AS entry_time
FROM dws_agri_harvest_daily
WHERE source_type = 'manual_entry'
  AND updated_at >= NOW() - INTERVAL '7 days'
ORDER BY updated_at DESC;
